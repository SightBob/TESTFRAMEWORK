6.	ความสำคัญและที่มาของปัญหา
	ปัจจุบัน ข้อมูลของผู้ที่ขอทำ EC หรือทำวิจัยในมนุษย์ถูกจัดเก็บในรูปแบบของไฟล์ Excel ซึ่งทำให้การบริหารจัดการข้อมูลเป็นเรื่องยุ่งยาก ไม่สะดวกต่อการค้นหาและเรียกดูข้อมูล อีกทั้งการใช้ไฟล์ Excel ยังมีข้อจำกัดในด้านการเข้าถึงและการบริหารจัดการข้อมูลร่วมกัน โดยเฉพาะอย่างยิ่งเมื่อมีหลายฝ่ายที่เกี่ยวข้องต้องใช้งานข้อมูลพร้อมกัน นอกจากนี้ ยังมีความเสี่ยงสูงต่อการสูญหายของข้อมูลหรือเกิดข้อผิดพลาดจากการคัดลอก แก้ไข และบันทึกข้อมูลหลายครั้ง ซึ่งอาจส่งผลกระทบต่อความถูกต้องและความสมบูรณ์ของข้อมูล

	จากการวิเคราะห์ระบบปัจจุบัน พบปัญหาสำคัญคือการแสดงข้อมูลนักวิจัยไม่ครบถ้วน โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title) ที่ไม่ถูกบันทึกเข้าสู่ฐานข้อมูลเมื่อมีการแก้ไข เนื่องจากขาดเมธอด POST สำหรับ EditInternal และ EditExternal ใน ResearcherController ทำให้ข้อมูลที่แก้ไขไม่ถูกบันทึกกลับเข้าสู่ฐานข้อมูล

	เพื่อแก้ไขปัญหาดังกล่าว การพัฒนาเว็บแอปพลิเคชันด้วย ASP.NET MVC เพื่อเป็นฐานข้อมูลกลางจึงเป็นแนวทางที่เหมาะสมและมีประสิทธิภาพ ระบบนี้ช่วยให้สามารถจัดเก็บ ค้นหา และเรียกดูข้อมูลได้อย่างเป็นระบบ ลดข้อผิดพลาดที่เกิดจากการใช้ไฟล์ Excel และเพิ่มความสะดวกให้กับผู้ใช้งาน โดยเฉพาะการค้นหาข้อมูลได้ในไม่กี่วินาที ผ่านฟังก์ชันการค้นหาอัจฉริยะที่สามารถกรองข้อมูลตาม ชื่อโครงการ ผู้วิจัย หรือวันที่ดำเนินการ ทำให้ผู้ใช้ไม่ต้องเสียเวลาค้นหาข้อมูลจากหลายไฟล์ นอกจากนี้ ระบบยังรองรับการจัดเก็บข้อมูลเป็นมาตรฐาน

7.	วัตถุประสงค์
	7.1 พัฒนาเว็บแอปพลิเคชัน ASP.NET MVC เพื่อจัดการข้อมูล EC และงานวิจัยในมนุษย์อย่างเป็นระบบ
โดยออกแบบระบบให้รวมข้อมูลโครงการวิจัย รายละเอียดผู้วิจัย และเอกสารจริยธรรม (EC) เข้าไว้ในแพลตฟอร์มเดียว เพื่อแก้ปัญหาการกระจายตัวของข้อมูลในไฟล์ Excel ที่จัดการยากและเสี่ยงต่อการสูญหาย พร้อมเพิ่มฟังก์ชันค้นหาที่กรองข้อมูลตามชื่อโครงการ ผู้วิจัย หรือวันที่ ช่วยลดเวลาการเข้าถึงข้อมูลจากหลายชั่วโมงเหลือไม่กี่วินาที

	7.2 แก้ไขปัญหาการบันทึกข้อมูลนักวิจัยไม่ครบถ้วน โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title) ด้วยการพัฒนาเมธอด POST สำหรับ EditInternal และ EditExternal ใน ResearcherController เพื่อให้ข้อมูลที่แก้ไขถูกบันทึกกลับเข้าสู่ฐานข้อมูลอย่างถูกต้องและครบถ้วน

	7.3 เพิ่มประสิทธิภาพการบริหารข้อมูลผ่านระบบอัตโนมัติและติดตามความคืบหน้างานวิจัยแบบ Real-Time โดยนำระบบตรวจสอบความถูกต้องของข้อมูล (Data Validation) มาใช้ลดข้อผิดพลาดจากการบันทึกซ้ำซ้อน และออกแบบแดชบอร์ดแสดงสถานะโครงการแบบเรียลไทม์ พร้อมการแจ้งเตือนกำหนดส่งเอกสารสำคัญ เพื่อเพิ่มความแม่นยำในการตัดสินใจของผู้เกี่ยวข้อง

8.	ขอบเขตงาน
8.1 ขอบเขตด้านการพัฒนาระบบและฐานข้อมูล
•	พัฒนาเว็บแอปพลิเคชัน ASP.NET MVC สำหรับจัดเก็บข้อมูล EC และงานวิจัยในมนุษย์
ระบบถูกออกแบบให้รองรับการบันทึกข้อมูลโครงการวิจัย รายละเอียดผู้วิจัย (ทั้งภายในและภายนอก) เอกสารประกอบการขอ EC และสถานะการดำเนินงาน โดยใช้ระบบความปลอดภัยแบบ Role-Based Access Control เพื่อควบคุมสิทธิ์การเข้าถึงข้อมูล
•	ออกแบบระบบฐานข้อมูล SQL Server เชิงสัมพันธ์ (Relational Database)
รองรับการจัดเก็บข้อมูลแบบโครงสร้าง (Structured Data) เช่น ชื่อโครงการ ประเภทการวิจัย วันที่ดำเนินการ และสถานะ EC พร้อมเชื่อมโยงกับเอกสารดิจิทัล (PDF/Word) เพื่อความสะดวกในการเรียกใช้และตรวจสอบ
•	พัฒนาเมธอด POST สำหรับ EditInternal และ EditExternal ใน ResearcherController
เพื่อแก้ไขปัญหาการบันทึกข้อมูลนักวิจัยไม่ครบถ้วน โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title) ให้สามารถบันทึกข้อมูลกลับเข้าสู่ฐานข้อมูลได้อย่างถูกต้องและครบถ้วน

8.2 ขอบเขตด้านฟังก์ชันและแดชบอร์ดติดตามผล
•	พัฒนาฟังก์ชันค้นหาและกรองข้อมูลอัจฉริยะ
ผู้ใช้สามารถค้นหาข้อมูลผ่านเงื่อนไข เช่น ชื่อโครงการ ผู้วิจัย หรือวันที่ โดยใช้เทคโนโลยี Full-Text Search เพื่อเพิ่มความแม่นยำ และกรองข้อมูลตามหมวดหมู่ เช่น สถานะ EC หรือประเภทการวิจัย 
•	ออกแบบแดชบอร์ดแสดงภาพรวมงานวิจัยแบบ Real-Time
แสดงสถิติโครงการวิจัยที่กำลังดำเนินการ/เสร็จสิ้น กราฟความคืบหน้าแต่ละขั้นตอน และการแจ้งเตือนอัตโนมัติเมื่อมีกำหนดส่งเอกสารสำคัญ เพื่อสนับสนุนการตัดสินใจของผู้เกี่ยวข้อง

9. ระเบียบวิธีวิจัย
งานวิจัยนี้ใช้ระเบียบวิธีการพัฒนาระบบ (System Development Methodology) แบบ Agile เพื่อออกแบบและพัฒนาเว็บแอปพลิเคชัน ASP.NET MVC สำหรับจัดการข้อมูล EC และงานวิจัยในมนุษย์
9.1 การออกแบบระบบ (System Design)
•	วิเคราะห์ความต้องการผู้ใช้
สัมภาษณ์ผู้เกี่ยวข้องหลัก 3 กลุ่ม (นักวิจัย เลขานุการ EC ผู้ตรวจสอบจริยธรรม) เพื่อกำหนดฟังก์ชันหลัก เช่น การบันทึกข้อมูล การค้นหาอัจฉริยะ และแดชบอร์ดติดตามงานวิจัย 
•	ออกแบบฐานข้อมูลเชิงสัมพันธ์ (Relational Database)
ใช้แบบจำลอง Entity-Relationship Diagram (ERD) เพื่อกำหนดโครงสร้างข้อมูล เช่น ตารางโครงการวิจัย ผู้วิจัย เอกสาร EC และสถานะการดำเนินงาน เพื่อป้องกันข้อผิดพลาดจากการจัดเก็บข้อมูลซ้ำซ้อน

9.2	การพัฒนาระบบ (System Development) เทคโนโลยีที่ใช้
•	Frontend & Backend : ASP.NET MVC 5
สร้างอินเทอร์เฟซผู้ใช้แบบ Responsive Design พร้อมฟังก์ชันค้นหาอัจฉริยะ (Full-Text Search) และการแสดงผลข้อมูลแบบ Real-Time การบันทึกข้อมูล การตรวจสอบสิทธิ์ (Authentication) และการเชื่อมโยงกับ SQL Server
•	ฐานข้อมูล : Microsoft SQL Server
จัดเก็บข้อมูลแบบโครงสร้าง (Structured Data) และไฟล์เอกสาร
•	การพัฒนาเมธอด POST สำหรับ EditInternal และ EditExternal
เพิ่มเมธอด POST ใน ResearcherController เพื่อรับข้อมูลจากฟอร์มแก้ไขและบันทึกลงฐานข้อมูล โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title) ที่มีปัญหาในระบบปัจจุบัน

9.3 การทดสอบระบบ (System Testing)
•	การทดสอบหน่วย (Unit Testing)
ตรวจสอบการทำงานของฟังก์ชันแต่ละส่วน เช่น ระบบค้นหา (Full-Text Search) และการอัปโหลดเอกสาร โดยเฉพาะการทดสอบเมธอด POST สำหรับ EditInternal และ EditExternal ว่าสามารถบันทึกข้อมูลได้อย่างถูกต้องและครบถ้วน
•	การทดสอบการใช้งานจริง (User Acceptance Testing)
ให้ผู้ใช้กลุ่มตัวอย่างทดลองใช้ระบบและประเมินผลผ่านแบบสอบถาม เพื่อปรับปรุงความสะดวกในการใช้งาน

9.4 การวิเคราะห์และประเมินผล (Analysis & Evaluation)
•	การวิเคราะห์ประสิทธิภาพ
เปรียบเทียบเวลาการค้นหาข้อมูลระหว่างระบบเดิม (Excel) และระบบใหม่ เพื่อยืนยันการลดขั้นตอนการทำงาน
•	การประเมินความเสี่ยง
ตรวจสอบความเสี่ยงด้านความปลอดภัยของข้อมูลตามมาตรฐานจริยธรรมการวิจัยในมนุษย์

9.5 ข้อพิจารณาด้านจริยธรรม (Ethical Considerations)
•	ออกแบบระบบให้สอดคล้องกับหลักจริยธรรมการวิจัยในมนุษย์ โดยป้องกันการเข้าถึงข้อมูลส่วนบุคคลของผู้วิจัยและอาสาสมัครที่ไม่จำเป็น
•	จัดทำคู่มือการใช้งานและจัดฝึกอบรมผู้ใช้ เพื่อลดความเสี่ยงจากการใช้งานผิดวิธี

10.	แผนการดำเนินงานตลอดโครงการ
1. ศึกษาความต้องการและปัญหาของระบบปัจจุบัน (2 สัปดาห์)
   - วิเคราะห์ปัญหาการบันทึกข้อมูลนักวิจัยไม่ครบถ้วน โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title)
   - สัมภาษณ์ผู้ใช้งานระบบเพื่อรวบรวมความต้องการเพิ่มเติม

2. ออกแบบระบบฐานข้อมูลและโครงสร้างเว็บแอปพลิเคชัน (3 สัปดาห์)
   - ออกแบบ Entity-Relationship Diagram (ERD)
   - ออกแบบส่วนติดต่อผู้ใช้ (User Interface) ด้วย ASP.NET MVC

3. พัฒนาเว็บแอปพลิเคชันต้นแบบ (6 สัปดาห์)
   - พัฒนาเมธอด POST สำหรับ EditInternal และ EditExternal ใน ResearcherController
   - พัฒนาระบบค้นหาและกรองข้อมูลอัจฉริยะ
   - พัฒนาแดชบอร์ดแสดงภาพรวมงานวิจัยแบบ Real-Time

4. ทดสอบระบบและปรับปรุง (3 สัปดาห์)
   - ทดสอบการทำงานของเมธอด POST สำหรับ EditInternal และ EditExternal
   - ทดสอบการค้นหาและกรองข้อมูล
   - ปรับปรุงระบบตามผลการทดสอบ

5. นำระบบไปใช้งานและเก็บฟีดแบค (2 สัปดาห์)
   - ฝึกอบรมผู้ใช้งานระบบ

12.	เป้าหมายของผลลัพท์ (Outcome) และตัวชี้วัด
1. ระบบสามารถจัดเก็บและค้นหาข้อมูลได้อย่างรวดเร็ว (เวลาค้นหา < 3 วินาที)
2. ข้อมูลนักวิจัยถูกบันทึกอย่างครบถ้วน โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title) ที่มีปัญหาในระบบปัจจุบัน
3. ลดข้อผิดพลาดจากการใช้ Excel ในการจัดเก็บข้อมูลได้อย่างน้อย 80% 
4. ผู้ใช้งานมีความพึงพอใจต่อระบบไม่น้อยกว่า 85% จากแบบสอบถาม

13.	ประโยชน์ที่คาดว่าจะได้รับ
1. ลดระยะเวลาในการค้นหาข้อมูลและเพิ่มความถูกต้องของข้อมูลนักวิจัย โดยเฉพาะข้อมูลคำนำหน้าชื่อ (Title)
2. ลดข้อผิดพลาดในการจัดการข้อมูลผ่านการพัฒนาเมธอด POST สำหรับ EditInternal และ EditExternal
3. เพิ่มความสะดวกในการจัดการข้อมูลสำหรับผู้ดูแลระบบผ่านอินเทอร์เฟซที่ใช้งานง่าย
4. สนับสนุนการดำเนินการวิจัยให้มีความคล่องตัวมากขึ้นด้วยระบบติดตามความคืบหน้าแบบ Real-Time

14.	การนำผลงานไปใช้ประโยชน์
1. ใช้เป็นระบบมาตรฐานในการจัดเก็บข้อมูลผู้ขอทำ EC หรือทำวิจัยในมนุษย์ของโรงพยาบาลมหาวิทยาลัยเทคโนโลยีสุรนารี
2. เป็นต้นแบบสำหรับการพัฒนาระบบจัดเก็บข้อมูลวิจัยในองค์กรหรือหน่วยงานอื่น ๆ
3. ขยายระบบให้รองรับโครงการวิจัยประเภทอื่นในอนาคต
